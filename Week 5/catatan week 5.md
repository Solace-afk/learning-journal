# Jurnal Belajar: Minggu 5

**Topik:** Dynamic Analysis, Debugging, dan Observasi Runtime

## Apa yang Saya Pelajari
Analisis Dinamis, yaitu sebuah proses membedah perangkat lunak dengan cara menjalankannya secara langsung di dalam sebuah lingkungan yang terkontrol. Berbeda dengan analisis statis yang hanya membaca kode, teknik ini berfokus pada observasi nyata terhadap perilaku program saat dieksekusi (*runtime*). Snalisis ini sangat penting untuk menangani kode yang terenkripsi atau disamarkan (*obfuscated*), melihat interaksinya dengan sistem operasi, serta memverifikasi kerentanan yang sebelumnya mungkin telah ditemukan pada tahap statis. Untuk melakukan hal tersebut dengan aman, diperlukan persiapan lingkungan lab seperti menggunakan *Virtual Machines* (VM), *Sandbox* seperti Cuckoo untuk otomasi analisis, serta alat *Snapshot Management* guna mengembalikan kondisi sistem ke titik bersih semula. Alat utama yang dibahas dalam analisis ini adalah *Debugger* (seperti x64dbg untuk Windows, GDB untuk Linux, dan WinDbg untuk kernel Microsoft) yang berfungsi untuk menghentikan eksekusi dan memeriksa memori. Terdapat beberapa mekanisme yang ada di *Dynamic Analysis* seperti mekanisme *debugging* mulai dari *Attaching* (menyambung ke proses berjalan), *Launching* (menjalankan program baru), hingga *Stepping* (menjalankan instruksi demi instruksi).

## Apa yang Saya Pahami
Terdapat fungsi esensial dari *Breakpoints* untuk menghentikan program di titik tertentu, yang terbagi menjadi tipe *Software* (mengganti instruksi dengan nilai INT 3), *Hardware* (menggunakan register CPU), dan *Memory* (berdasarkan akses baca/tulis memori). Ketika program berhenti, kita dapat langsung menginspeksi nilai variabel pada *Register* CPU dan melacak urutan pemanggilan fungsi beserta variabel lokalnya melalui analisis *Stack* dan *Call Stack*. Banyak program berbahaya berinteraksi dengan OS melalui panggilan API, sehingga teknik *API Hooking* sangat berguna untuk mencegat dan mencatat niat asli dari program tersebut (seperti fungsi `CreateFile` atau `InternetOpen`). Selain memori, analisis dinamis juga mewajibkan kita untuk memantau lalu lintas jaringan menggunakan alat seperti Wireshark untuk melacak transmisi data, serta menggunakan Procmon (*Process Monitor*) untuk mencatat pembuatan file baru atau modifikasi pada *Registry* Windows secara *real-time*.

## Apa yang Masih Membingungkan
Meskipun saya sedikit memahami konsep dasar dari teknik perlindungan perangkat lunak, materi mengenai *Anti-Debugging* masih memberikan saya banyak kebingungan. Seperti pengembang dapat menanamkan trik seperti pengecekan *timing* eksekusi atau deteksi *breakpoint* untuk mengetahui apakah aplikasinya sedang dibedah, namun saya masih kesulitan cara melakukan *bypass* secara manual. Instruksi seperti memanipulasi *flag* atau melakukan *patching* untuk mengubah alur eksekusi JMP/CMP masih terasa rumit.

## Kesimpulan Pribadi
Kesimpulan yang saya dapatkan: analisis dinamis adalah jembatan vital untuk memahami aspek operasional sebenarnya dari sebuah perangkat lunak. Terutama dalam menganalisis sebuah *malware*, observasi *runtime* ini adalah satu-satunya cara pasti untuk mengungkap alamat *Command and Control* (C2), mendeteksi mekanisme persistensi, dan melihat secara empiris jenis data apa yang sedang dicuri. Saya juga memetik pelajaran krusial mengenai disiplin keamanan dan etika laboratorium; setiap kali melakukan bedah dinamis pada program berbahaya, hal tersebut mutlak dilakukan di dalam jaringan yang terisolasi (*Host-only*) agar infeksi tidak menyebar ke sistem utama lokal.
